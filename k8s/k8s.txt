K8S позволяет:
автоматически развертывать приложение на разных серверах
распределять нагрузку по нескольким серверам
автоматически масштабировать развернутые приложения
мониторинг и проверка работоспособности контейнера
замена нерабочих контейнеров


Среды запуска контейнеров:
docker, cri-o, containerd


Термины K8S:
pod - запрос на запуск одного или более контейнеров на одном узле.
pod содержит в себе 1 или более контейнеров, общий том, общий ip адрес

кластер K8S
кластер содержит узлы. каждый узел - сервер (node)
узлы могут быть как физические, так и виртуальные
внутри узла лежит pod
узлы имеют роли. главный узел управляет рабочими узлами


Сервисы K8S:
Главный и рабочий узлы содержат в себе: 
Container runtime (сервис для создания контрейнера),
kube-proxy(сетевые ресурсы),
kublet(коммуникация между разными узлами) (взаимодействует с API service)

Главный сервис содержит:
API Server 
Schelduler (планировщик)
Kube controller manager (сервис для контроля подов внутри кластера)
Cloud controller manager (взаимодействие с менеджером на стороне провайдера)
etcd (логгирование информации)
dns (получение доменных имен, как понятно из названия)


KUBECTL:
kubectl взаимдействует с главным узлом через api service. в качестве протокола
используется https.


при запуске контейнера внутри k8s запускаются два контейнера: 
основной и служебный. служебный контейнер резервирует ресурсы 
для работы основного контейнера
